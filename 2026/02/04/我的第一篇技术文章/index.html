搭建个人技术博客

为什么要搭建个人博客

在学习编程的过程中，我发现记录和分享是非常重要的。通过写博客，我可以：

  1. 巩固知识：通过写作整理思路，加深理解
  2. 建立知识体系：将零散的知识点系统化
  3. 分享交流：与其他开发者交流经验
  4. 长期积累：为自己的学习之路留下记录

技术选型

我选择了 Hexo + GitHub Pages 的方案：

  • Hexo：快速、简洁且高效的静态博客框架
  • GitHub Pages：免费的静态网站托管服务
  • Butterfly 主题：美观、功能丰富的 Hexo 主题

搭建过程

整个搭建过程非常简单：

  1. 安装 Node.js 和 Git
  2. 安装 Hexo CLI
  3. 初始化博客
  4. 安装主题
  5. 配置 GitHub Pages
  6. 部署上线

总共耗时约 1.5 小时，后续维护成本几乎为零。

Markdown 写作体验

使用 Markdown 写作非常方便，支持：

  • 代码高亮：js ...
  • 表格：
    列1 列2
    A B
  • 列表：有序列表和无序列表
  • 引用：> 引用内容

总结

这是我博客的第一篇文章。后续我会记录学习过程中的各种技术笔记，包括但不限于：

  • 前端开发
  • 后端开发
  • 工具使用技巧
  • 项目实战经验

如果你也想搭建个人博客，推荐从 Hexo 开始！
